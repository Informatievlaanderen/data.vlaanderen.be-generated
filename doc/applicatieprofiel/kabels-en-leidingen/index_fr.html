Dépendances

Ce document décrit un profil d'application, en l'espèce OSLO Câbles et Conduites (Profil d'application). Cet profil d'application indique comment le modèle du domaine correspondant peut être appliqué dans la pratique. Les contraintes (cardinalité, listes de codes) y sont expliquées et les termes correspondants (RDF) sont répertoriés.

Résumé

Le profil d'application OSLO Câbles et Conduites montre comment les termes du vocabulaire associé doivent être utilisés pour décrire les câbles et conduites.

Ces éléments peuvent être situés à la fois au-dessus et en dessous du sol. En ce qui concerne les câbles et conduites souterrains, la description a pour objectif que tous les éléments soient présents afin de limiter les dommages lors des travaux d'excavation.

Le modèle s'appuie sur les INSPIRE Utility Services et sur les modèles sur lesquels ce modèle est basé, principalement INSPIRE Networks.

Les services tels que les câbles et conduites sont conçus comme un réseau (un UtilityNetwork) dans lequel ils forment des éléments linéaires, connectés par des objets ponctuels tels que des splitters, des connecteurs, des connexions montées dans des armoires, des tours, des poteaux, etc.

Les câbles et conduites sont des collections de tronçons dans le réseau, des Ensemble de Tronçons de Service composés de Séquences de Tronçons de Service ou de Tronçons. Les principaux types d'ensemble de tronçons sont les Câbles, les Conduites et les Canalisations. Les Conduites et les Canalisations sont des conduites : ils transportent des substances telles que l'eau, les eaux usées, le gaz ou contiennent des Câbles ou d'autres Conduites ou Canalisations.

La classe des Câbles est divisée en Câbles Électriques et Câbles de Télécommunications. Dans les Conduites, une distinction est faite entre les Conduites d'Hydrocarbures, les Conduites d'Eaux Usées, les Conduites d'Eau et les Conduites Thermiques.

La géométrie des Ensemble de Tronçons de Service des services correspond à celle des Séquences de Tronçons de Service ou des tronçons constitutifs. Dans le cas d'une séquence de tronçons, il s'agit de la géométrie des tronçons formant la séquence, mais dans une direction particulière (important, par exemple, pour le drainage). Pour les tronçons, il s'agit de la géométrie de la ligne médiane.

Les objets pouvant être représentés sous forme de point qui réalisent les connexions dans le réseau (par exemple, raccords, connexions, pompes, compteurs) sont les nœuds du réseau, les soi-disant Noeuds de Service. Ils peuvent être autonomes mais sont généralement montés sur des objets plus volumineux tels que des poteaux ou des tours, ou sont placés dans des armoires sur la chaussée. Nous les appelons des Conteneurs de Nœuds de Service.

Certains Eléments de Service forment ensemble une installation ou une installation qui peut être décrite séparément comme un Complexe d'Activités.

Le modèle comprend diverses classes et types de données de support, notamment pour compléter les informations du réseau avec des informations topographiques (classe Elément Topographique) ou des zones où l'excavation n'est pas autorisée (classe Zone Protégée) ou pour annoter les objets (classe Annotation).

La géométrie des câbles et conduites et des nœuds dans le réseau peut être décrite en 2D ou en 2.5D.

Si le 2.5D n'est pas possible ou est insuffisante, des informations supplémentaires sur la Position Verticale peuvent être fournies, à la fois sous forme relative (par rapport à une Surface de Référence) et absolue (Position Directe dans un système de coordonnées verticales tel que le TAW). Ces informations sont importantes pour éviter les dommages lors des travaux d'excavation.

Accessoire

Description
Partie physique du Réseau de Service pouvant être représenté sous forme d'un point.
Usage
Il peut être autonome en tant que Noeud de Service, mais il est souvent monté sur un support (le Conteneur du Noeud Service). Par ex.: trappe d'inspection, bouche d'incendie, splitter, jauge, pompe, etc.
Sous-classe de
Nutsvoorzieningknoop
Propriétés
Pour cette entité, les propriétés suivantes sont définies: hauteur, type d'accessoire.
Propriété Type attendu Cardinalité Description Usage Codelist

type d'accessoire Type d'Accessoire 1..* Type d'Accessoire.

Adresse

Description
Informations permettant de se référer de manière structurée et unique à une unité de bâtiments, un poste d’amarrage, un emplacement ou une parcelle sur la base d'un nom de commune, d'un nom de rue, d'un numéro de maison et éventuellement d'un numéro de boîte et d'un code postal.
Usage
La définition se réfère aux adresses belges. Dans d'autres pays, une adresse peut éventuellement se référer à d'autres objets géographiques (pas une unité de bâtiment ou une parcelle) ou consister en d'autres désignations (différentes du numéro de maison et du numéro de boîte) ou en des éléments autres que le nom de la rue, le nom de la commune ou le code postal. D'autres indications d'adresse que le numéro de maison ou le numéro de boîte (par ex., désignation de l'entrée, désignation de l'escalier, désignation de l'étage) ou un nom de bâtiment ou de chambre sont possibles grâce à une association avec l'extension de l'adresse. L'échange d'adresses étrangères doit être effectué à l'aide de l'objet représentation d'adresse.
Propriétés
Pour cette entité, aucune propriété n'est définie.

Agent

Description
Quelqu'un ou quelque chose qui peut agir ou produire un effet.
Propriétés
Pour cette entité, aucune propriété n'est définie.

Annotation

Description
Une courte explication ou une note ajoutée à un Objet.
Usage
Dans ce contexte, il s'agit généralement d'une Indication de mesure sur un plan de câble ou de conduite, par ex. la distance d'un câble par rapport à une façade. Peut se composer d'éléments (par ex.: une ligne de dimensionnement, une ligne de guidage de dimensionnement, deux flèches) qui sont également des Annotations en soi.
Propriétés
Pour cette entité, les propriétés suivantes sont définies: angle de rotation, associé à, ensemble, localisation, partie, texte, type de l'annotation.
Propriété Type attendu Cardinalité Description Usage Codelist
angle de rotation Taille 0..1 Angle de rotation de l'Annotation à l'affichage. Généralement pour aligner l'Annotation à l'Objet auquel elle se rapporte, par ex. parallèlement au câble.
associé à Objet 1 Objet auquel l'Annotation se rapporte Par ex.: un câble.
ensemble Annotation 0..* Annotation dont l'Annotation fait partie. Par ex.: l’Indication de mesure à laquelle la ligne de dimensionnement appartient.
localisation Géométrie 1 Géométrie de l'Annotation. Par ex.: l'emplacement et la forme d'une Indication de mesure.
partie Annotation 0..* Annotation faisant partie de l'Annotation. Par ex.: la ligne de dimensionnement dans le cas d'une Indication de mesure composée de deux flèches et d'une ligne de guidage de dimensionnement.
texte LangString 0..* Texte de l'Annotation. Par ex.: la taille dans le cas d'une Indication de mesure.
type de l'annotation Type d'annotation 1 Nature de l'Annotation.

Courbe

Description
Primitive géométrique unidimensionnelle.
Usage
La dimension doit être comprise dans un sens mathématique : le nombre minimum de coordonnées nécessaires pour représenter tout point faisant partie de la primitive. Dans le système de coordonnées formé par une Courbe elle-même, 1 coordonnée suffit pour localiser un point sur la Courbe, par ex. la distance le long de la Courbe à partir du point de départ. Une Courbe a une longueur mais pas de largeur ni de hauteur, ce qui ne l'empêche pas de serpenter dans l'espace 3D. Les Courbes sont généralement représentées par des polylignes, qui sont des ensembles de points consécutifs où les points intermédiaires de la Courbe sont obtenus par interpolation. Lorsque la méthode d'interpolation est linéaire, vous obtenez une Ligne Polygonale (une ligne angulaire reliant des points successifs par une ligne droite). D'autres méthodes d'interpolation donnent lieu à des arcs de cercle, des cubic splines, des courbes de Bézier, etc. (dont les points de contrôle ne sont pas nécessairement situés sur la Courbe). Une Courbe est délimitée par ses points de départ et d'arrivée.
Sous-classe de

Point

Description
Primitive géométrique zéro-dimensionnelle.
Usage
La dimension doit être comprise dans un sens mathématique : le nombre minimum de coordonnées nécessaires pour représenter tout point faisant partie de la primitive. Dans le système de coordonnées formé par un Point lui-même, aucune coordonnée n'est nécessaire, puisqu'il n'y a qu'un seul point. Un point n'a donc ni largeur, ni longueur, ni profondeur. ATTENTION : cela ne signifie pas qu'un Point ne peut pas avoir de coordonnées dans un autre système de coordonnées, par ex. des coordonnées xyz dans un système de coordonnées à 3 axes orthogonaux.

Position Directe

Description
Contient les coordonnées d'une position dans un système de référence de coordonnées.
Usage
Généralement utilisé pour décrire la géométrie d'un objet spatial ou les primitives géométriques (par exemple point, ligne, plan) qui composent cet objet. Cependant, un DirectPosition peut également être autonome pour décrire une position dans l'espace. La position est donnée dans un système de référence de coordonnées. Ce système de référence est explicitement spécifié avec DirectPosition, sauf s'il fait partie d'un objet spatial (par exemple une primitive géométrique) pour lequel un système de référence est déjà indiqué. Pour information : un système de référence de coordonnées diffère d'un système de référence spatiale indirecte dans la mesure où les positions sont spécifiées directement par des coordonnées plutôt qu'indirectement par une adresse ou un nom de lieu.
Propriétés
