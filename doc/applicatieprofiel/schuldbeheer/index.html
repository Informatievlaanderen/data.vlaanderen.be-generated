Afhankelijkheden

Dit document beschrijft een applicatieprofiel, in dit geval OSLO Schuldbeheer - Applicatieprofiel. Dit applicatieprofiel beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Entiteiten

Aanmaning

Beschrijving
Schriftelijk berichtgeving van een schuldeiser naar een schuldenaar om deze schuldenaar te herinneren aan een vordering die niet is voldaan terwijl de vervaldatum van deze vordering reeds verstreken is.  
Subklasse van
Activiteit
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: aanmaningsdatum.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
aanmaningsdatum Date 1 Datum waarop de aanmaning werd verstuurd. 

Activiteit

Beschrijving
Handeling van bepaalde duur op of met entiteiten.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: einddatum, genereert, heeft deelnemer, startdatum, werd gestart door.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
einddatum DateTime 1 De einddatum van de activiteit.
genereert Stuk 0..* Een activiteit kan een stuk genereren.
heeft deelnemer Agent (in rol) 0..* Persoon of organisatie die binnen een bepaalde rol een verantwoordelijkheid opneemt voor deze activiteit.
startdatum DateTime 1 De startdatum van de activiteit.
werd gestart door Agent 0..1 Van een start is sprake wanneer een activiteit geacht wordt te zijn gestart. Een start kan verwijzen naar een entiteit, ook wel een trigger genoemd, die de activiteit heeft geïntitieerd.

Afbetalingsplan

Beschrijving
Een gestructureerd schema dat wordt gebruikt om een schuld af te lossen. 
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: vordering.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
vordering Vordering 1..* Verwijzing naar een vordering waarop het afbetalingsplan van toepassing is

Agent

Beschrijving
Een agent is iets dat een of andere vorm van verantwoordelijkheid draagt voor een activiteit die plaatsvindt, voor het bestaan van een entiteit of voor de activiteit van een andere agent.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: handelt in opdracht van.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
handelt in opdracht van Agent 0..* Een agent kan handelen in opdracht van een andere agent.

Agent (in rol)

Beschrijving
Een toewijzing van verantwoordelijkheid aan een agent voor een activiteit, wat aangeeft dat de agent een rol in de activiteit had.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: is rolspecialisatie van, rol.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
is rolspecialisatie van Agent 1 Duidt de agent aan waarvan deze entiteit een rol toewijst.
rol Rol 1 De functie van een agent met betrekking tot een activiteit.

Betaling

Beschrijving
De overdracht van geld van één of meerdere partij(en) naar een andere partij. 
Subklasse van
Activiteit
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: bestemming, betaald bedrag, betalingsdatum, identificator, toerekening van Betaling.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
bestemming Vordering 0..* Verwijzing naar de vordering die de betaler wenst af te lossen met de betaling.
betaald bedrag Geldbedrag 1 Het geldbedrag dat werd betaald.
betalingsdatum Date 1 Datum waarop het overgedragen geld werd ontvangen door de ontvanger. 
identificator Identificator 1 Een identificator voor het betaling
toerekening van Betaling Toerekening van Betaling 0..* Verwijzing naar de activiteit die (een deel van) een betaald bedrag toewijst aan een vordering.

Toerekening van Betaling

Beschrijving
Het toewijzen van een betaald geldbedrag aan een vordering. 
Subklasse van
Activiteit
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: betaling, toegerekend bedrag.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst

toegerekend bedrag Geldbedrag 1 Het geldbedrag dat werd toegerekend aan een vordering.

Vordering

Beschrijving
Aanspraak van een partij (de schuldeiser) op een bepaalde prestatie vanwege een andere partij (de schuldenaar).
Gebruik
Voor de doeleinden van dit applicatieprofiel is de opgelegde prestatie altijd de betaling van een geldsom. 
Subklasse van
Activiteit
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: aanmaning, afbetalingsplan, betwisting, grondslag, hoofdsom, identificator, isGerelateerdMet, kwijtschelding, oninbaarheidsverklaring, toerekening van Betaling, uiterste betaaldatum, vorderingstype, wanbetalingskost.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
aanmaning Aanmaning 0..* Verwijzing naar een aanmaning met betrekking tot de vordering.
afbetalingsplan Afbetalingsplan 1 Verwijzing naar een afbetalingsplan dat voor de vordering geldt.
betwisting Betwisting 0..* Verwijzing naar een betwisting van de vordering.
grondslag Grondslag 1 Verwijzing naar de juridische basis voor de vordering
hoofdsom Geldbedrag 1 Het totale bedrag dat wordt gevorderd door de schuldeiser, exclusief eventuele wanbetalingskosten.
identificator Identificator 1 Een identificator voor de vordering
isGerelateerdMet Vordering 0..* Verwijzing naar een andere vordering die een verband heeft met de eerste vordering. Men kan beslissen om een nieuwe vordering aan te maken voor een wanbetalingskost, in plaats van de wanbetalingskost rechtstreeks aan de vordering toe te voeging.
kwijtschelding Kwijtschelding 0..* Verwijzing naar een kwijtschelding voor de vordering.
oninbaarheidsverklaring Oninbaarheidsverklaring 0..1 Verwijzing naar een oninbaarheidsverklaring voor de vordering.
toerekening van Betaling Toerekening van Betaling 0..* Het toewijzen van een betaald geldbedrag aan een vordering. 
uiterste betaaldatum Date 1 De laatst mogelijke datum waarop een betaling moet worden gedaan om te voorkomen dat er eventuele boetes, interesten of andere sancties worden opgelegd.
vorderingstype Vorderingscategorie 1 Verwijzing naar een vorderingstype.
wanbetalingskost Wanbetalingsvergoeding 0..* Verwijzing naar een wanbetalingskost.

Vorderingscategorie

Beschrijving
Een soort vordering met bepaalde kenmerken.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: fiscaal, type, verjaringstermijn.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
fiscaal Boolean 1 Laat toe om te specifiëren of het een fiscale of niet-fiscale vordering betreft.
type Vorderingstype 1 Het label van het vorderingstype Bijvoorbeeld schoolfactuur, GAS-boete
verjaringstermijn Duration 1 De verjaringstermijn die voor vorderingen van dit classificatie geldt.

Wanbetalingsvergoeding

Beschrijving
Vergoeding die in rekening worden gebracht als gevolg van een wanbetaling.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: aanmaning, bedrag, percentage, type.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
aanmaning Aanmaning 0..* Verwijzing naar een aanmaning die waarin de wanbetalingskost wordt gecommuniceerd aan de schuldenaar.
bedrag Geldbedrag 1 De omvang van de wanbetalingsvergoeding kan worden bepaald door een bedrag.
percentage Percentage 1 De omvang van de wanbetalingsvergoeding kan worden bepaald als een percentage.
type 1 Laat toe om het type van de wanbetalingskost verder te specifieren Bijvoorbeeld verwijlinterest, aanmaningskost, verzendingskost
